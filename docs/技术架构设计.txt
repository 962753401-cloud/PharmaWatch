这份AI云眼医保基金监管系统（离线审计版）的完整技术架构设计方案，专为事后非线性数据核查、高吞吐审计、强证据链闭环的政务监管场景打造。

系统核心采用“数据驱动倒查、本地高速粗筛、云端大模型精审、一案一册归档”的离线批处理架构。它彻底摆脱了实时视频监控流对高带宽和超高边缘算力的依赖，将大模型 API 的调用成本压缩了 90% 以上。

## 🛠️ 核心图层功能与组件细分

### 1. 历史数据源层（Data Sources Layer）

存储并供应事后审计所需的全部异构原始数据。

* **结算数据库**：存储定点医药机构的线上结算流水（包含药店编号、终端号、参保人ID、结算时间戳、药品及数量清单、金额）。
* **本地存储（NAS/网盘/硬盘组）**：存放药店依规录制并定期上载的音视频文件。视频格式统一为 H.264/MP4，音频格式统一为高保真 PCM/WAV。

### 2. 时空对齐与智能切片层（Smart Slicing Layer）

利用“数据先行”策略，解决全量视频无脑分析产生的算力灾难。

* **时间戳锚定索引**：系统批量提取可疑或抽查的医保结算流水，通过“药店编码+终端号+结算时间”动态计算出对应的音视频文件名。
* **FFmpeg 黄金片段裁剪**：自动调用 FFmpeg 离线无损裁剪脚本。以结算时间为中心，向前回溯 45 秒（捕捉挑选商品、递卡、沟通动作），向后延伸 15 秒（捕捉付款、找零、拿渡物品动作），切取 60 秒的“黄金核查片段”。

### 3. 本地高速批处理与特征粗筛层（Batch Filter Layer）

在本地服务器运行，目标是**榨干本地硬件，高速过滤掉绝对合规的画面，提取基础空间特征**。

| 技术组件 | 处理模式 | 核心稽查职责 |
| --- | --- | --- |
| **YOLO11** | Batch 批处理模式 (128 帧并发) | 本地无需受 30 FPS 物理帧率限制，以 5 倍速以上超高速扫描切片。主要识别画面中是否有卡片（`card`）、现金（`cash`）、人员（`person`）以及非药品包装。 |
| **Supervision (ByteTrack)** | 离线非线性轨迹关联 | 1. 自动映射预设的“收银交互多边形区域 (`PolygonZone`)”。<br>

<br>2. 统计 60 秒内该区域发生的交互特征。例如：若结算单显示有交易，但 Supervision 判定收银区**持卡数始终为 0 或人数为 0**，系统直接预警为“虚构医药服务（假刷卡）”，无需再调大模型。 |

### 4. 中枢调度与轻量化传输层（Orchestrator Layer）

整个离线系统的“大管家”，负责任务的高效排队分发与数据瘦身。

* **Celery + Redis 分布式队列**：将每个需要审计的“小票+视频”打包成一个独立 Job。支持多线程并发、断点续传。通常可配置为“夜间低峰期自动批量上云精审”。
* **视频极致压缩编码**：利用 FFmpeg 对 60 秒取证视频进行高倍率二次压制。**分辨率强转为 640×480（480P），帧率抽样降至 3 FPS**（每秒仅保留 3 关键帧）。60 秒视频压缩后仅包含 180 张画面，体积暴降至几百 KB，彻底打碎公网传输瓶颈。

### 5. 云端大模型多模态联合审判层（Cloud Audit Layer）

通过 HTTPS 接口异步调用通义千问官方 API，实现真正的**多模态融合交叉质检**。

* **Qwen ASR API (听觉还原)**：输入 60 秒音频，高精转写为带有词级时间戳（Word-level Timestamp）的文本片段，自动检索“换成别的”、“返点现”、“帮我家人刷”等敏感口语。
* **Qwen3-VL API (视听数终审)**：将【轻量化视频片段 + ASR 文本上下文 + 医保结算流水明细】打包为统一的多模态 Payload 发送给 API。大模型通过长视频时序理解能力，还原完整的违规证据链，并强制输出结构化 JSON 判定书。

## 💻 呈现层设计：医保“一案一册”智能办案看板

在制作给医保局演示的前端 Demo 时，核心技术呈现应围绕“非线性离线追溯”展开，实现以下三大亮点：

1. **“多模态时空对齐播放器（Timeline Syncer）”**：
稽查人员在列表中点击任意“疑似违规件”，进入详情页。播放器在播放 60 秒取证视频（画面上由 OpenCV 自动叠加 YOLO11 识别出的卡片、商品红框）的同时，右侧的语音转写文本会像歌词一样**逐字高亮滚动**。当播放到第 22 秒店员拿红参礼盒时，对应的敏感语音“换成红参礼盒呗”同时闪烁。
2. **“关键线索特征面板”**：
界面显著位置展示本地轻量感知过滤层提取的静态指标（例如：`[本地粗筛：画面中出现医保卡数量：3张]`），大模型终审报告以纯自然语言形式直接展示在右侧，排版如同真实的**医院病历或法院判决书**。
3. **“一键导出标准化稽查卷宗”**：
Demo 页面提供一个“导出卷宗（Zip）”按钮。导出的压缩包解压后包含：
* `evidence_video.mp4`：烧录有药店防伪水印、时间戳以及视觉 Bounding Box 的现场视频。
* `audit_report.pdf`：自动生成的正式医保稽查报告，内含违规定性、大模型评语、涉案转写文本，并预留稽查员签字栏。
